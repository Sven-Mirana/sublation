# 委托数据隔离事件 · 2026-05-27

## 事件

Hermes 准备通过 `delegate_task` 将任务委派给 Codex 时，在 `context` 字段中包含了本地 workspace 的完整摘要——文件路径、案件信息、目录结构、config 数据。

用户立即阻止：「不要把本地 workspace 摘要发给外部模型服务」。

## 根因

`delegate_task` 的 `context` 字段被当作自由格式的"背景信息"使用，Hermes 习惯性地把"所有可能相关的信息"塞进去，以帮助子代理减少来回询问。

这忽略了 `context` 是明文发送给外部模型（Codex/GPT-5.5）的——不在用户的直接授权范围内。

## 修正

已在 `skill-sublation` SKILL.md 中新增"协作安全：委托到外部模型的数据隔离"章节，作为三房协作的硬边界：

- `context` 只传任务描述 + 明确参数（候选ID、分支名）+ 抽象化的技术上下文
- 文件内容由 Codex 在自己 session 中自行读取，不提前推送
- 禁止传播：路径清单、文件正文、案件信息、workspace 摘要

## 教训

这是三房协作的安全红线。用户信任我们自主推进，但本地数据隔离是信任的前提——"自主"不等于"自由传数据给外部"。
